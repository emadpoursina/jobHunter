# Workflow principles

Decision log for jobHunter. Update when rules change.

## Role & migration

- **Target roles:** Backend software developer; full-stack when backend-heavy.
- **Migration path:** Skilled migration via job offer (employer sponsorship or points tied to employment).
- **Seniority:** Pick one band per sampling round (e.g. mid-level) and stay consistent across countries.

## Sampling (Phase 1)

- **Five priority countries** — maximum effort; still apply elsewhere at lower weight.
- **10–15 offers per priority country** — not 10 total across all countries.
- **Recency:** Prefer postings from the last ~90 days.
- **Sources:** Company careers pages, LinkedIn, reputable national boards.
- **Exclude:** Unrelated titles, intern/junior mismatch, duplicate reposts, unclear visa/sponsorship where that matters.

## Immigration (Phase 1)

For each priority country document:

- Typical visa route for software developers with a job offer
- Language requirements (official + practical)
- Whether employers commonly sponsor non-citizens
- Known blockers (local degree, local experience, salary thresholds)

## Applications (Phase 2)

- **Master profile first** — never edit generated CVs without syncing learnings back to master profile.
- **AI drafts, human approves** — 100% review before send; verify facts, tone, and role alignment.
- **Phase 2 ∥ Phase 3** — apply while upskilling; do not block applications on gap closure.
- **Feedback loop** — rejection reasons and interview notes update `skills/gap-report.md` and `learning/backlog.md`.

## Networking (Phase 0)

- Run in parallel with Phase 2 from day one.
- Log every outreach and link referrals to `applications/pipeline.md`.

## Metrics

- Review `metrics/by-country.md` weekly.
- If response rate stays low for a priority country after sufficient volume, re-check offer fit, CV tailoring, or deprioritize country.
